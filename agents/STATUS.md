- `transcriber.rs` — whisper-rs 0.13 wrapper, single-loaded `WhisperContext`, per-call `WhisperState`. `inflight: Mutex<()>` serialises live + final passes (whisper.cpp isn't reentrant across states). Energy gate at RMS ≥ 0.009 f32 (= ~300 on i16 / 32768) drops silent buffers before whisper sees them. 4-gram repetition check (`is_hallucination`) drops "thanks for watching" / "subtitles by..." artefacts. Sub-1 s buffers padded to 16 000 samples to dodge the spike-documented whisper.cpp empty-segment wall.
- `audio_capture.rs` — cpal `Stream` is `!Send` on Windows, so it's owned on a dedicated `voice-capture` thread that signals teardown via crossbeam. Always normalises to 16 kHz mono f32 (downmix + linear-resample in the data callback) so VAD + whisper never see device-specific rates. Handles F32, I16, U16 sample formats; rejects others with a clear `AppError`.
- `vad.rs` — `Vad` trait + always-available `EnergyVad` (RMS threshold) + feature-gated `SileroVad` (ONNX via `ort` 2.0-rc, behind `silero-vad` feature). `build()` factory returns `Box<dyn Vad>` and falls back to `EnergyVad` whenever the feature is off, the `.onnx` file is missing, or the runtime library can't be resolved — voice is never blocked by VAD-model issues.
- `pipeline.rs` — `VoiceService` orchestrator. Three concurrent actors per session: cpal callback thread, `voice-rolling` worker (polls buffer every 100 ms, applies min-speech 0.75 s / max-live 5.0 s / silence-wait 0.5 s timing + VAD silence check + 0.5 s overlap, mirrors `pipeline.py::_rolling_transcribe_loop`), and `voice-transcribe` worker that owns the channel + emits `voice:partial` (deduped against last 10 transcripts) / `voice:final` events. `stop()` snapshots the buffer, sends Final + Stop on the channel, blocks on the final result with a 30 s timeout, and emits `voice:state { idle }`.
- `text_injection.rs` — Win32 `SendInput` with `KEYEVENTF_UNICODE` per UTF-16 code unit + 5 ms inter-keystroke delay, port of `text_injection.py:40-87`. Non-Windows path is a no-op (logs only) so the rest of the subsystem stays cross-platform for `cargo test`.
- `model.rs` — first-run discovery + copy. Looks for `ggml-base.en-q5_0.bin` under `%APPDATA%/pixiis/models/whisper/`, falls back to `<exe>/resources/models/whisper/`, then `../resources/models/...` for dev runs. Same pattern for `silero_vad.onnx`. Returns the bundled path directly when the user dir isn't writable rather than failing.

`Cargo.toml`: added `whisper-rs 0.13`, `cpal 0.15`, `crossbeam-channel 0.5`, `dirs 5`. `ort 2.0-rc.10` + `ndarray 0.16` are optional and behind the `silero-vad` feature (default OFF — keeps `cargo build --release` from depending on the rc-churning `ort` API + `onnxruntime.dll` deployment). `windows` crate gains `Win32_UI_Input_KeyboardAndMouse` for `SendInput`.

`commands/voice.rs`: real impls for all six commands. `VoiceServiceSlot { service: Option<Arc<VoiceService>>, init_error: Option<String> }` so a missing/broken model surfaces as `AppError::Other("voice unavailable: …")` from `voice_start` rather than panicking at startup. `voice_stop` and `voice_get_devices` and `voice_inject_text` push their blocking work onto `tokio::task::spawn_blocking` so the async runtime stays responsive. `voice_get_transcript_log` returns `[]` when the service is down.

`lib.rs::run::setup` instantiates the service from `voice::model::ensure_default_whisper_model()` + `ensure_silero_model()` and `app.manage(Arc::new(VoiceServiceSlot { ... }))`. Whisper model load is synchronous in setup — adds 200–500 ms of startup latency on production NTFS (per spike RESULTS.md), acceptable for the MVP; Pane 5's Settings page can kick a background reload later.

`tauri.conf.json` `bundle.resources` now lists `../resources/models/whisper/*.bin` + `../resources/models/silero/*.onnx`. `resources/models/{whisper,silero}/README.md` documents the Hugging Face / GitHub URLs to fetch the weights — the actual binaries are not committed (Q5_0 base.en is ~31 MB, Silero ONNX is ~2 MB). The user (or a release script) drops them in before `npm run tauri build`.
